# YADRO Impulse 2026 — DevOps Infrastructure

Репозиторий содержит артефакты выполнения тестового задания для команды разработки внутренней инфраструктуры.

## Структура проекта

### Раздел 1: Python Script (`task1/`)
Скрипт для тестирования HTTP-ответов с использованием библиотеки `requests`.
*   **Особенность:** В связи с нестабильной работой целевого сервиса `httpstat.us` (ошибки `ERR_EMPTY_RESPONSE` со стороны провайдера), для обеспечения надежности тестирования скрипт использует функциональный аналог — `https://httpbin.org/status/`. 
*   **Реализация:** Логирует успешные ответы (коды 1xx-3xx). Для кодов 3xx отключен автоматический редирект. При получении 4xx и 5xx генерируется исключение `HTTPError`.

### Раздел 2: Docker (`task2/`)
Docker-образ для запуска скрипта.
*   **База:** `ubuntu:22.04`.
*   **Оптимизация:** Пакеты устанавливаются в одном слое, чтобы минимизировать размер образа. Установлена переменная окружения `PYTHONUNBUFFERED=1` для корректного и мгновенного логирования вывода скрипта в `docker logs`.

### Раздел 3: Ansible (`task3/`)
Ansible-плейбук для автоматизации полного цикла деплоя на удаленный сервер.
*   **Сценарий:** Плейбук устанавливает необходимые системные зависимости и Docker. Добавляет пользователя в группу `docker`, запускает службу.
*   **Доставка и запуск:** Реализовано копирование исходного кода из папки `task2` на целевой сервер, сборка Docker-образа на стороне сервера и запуск контейнера.
*   **Проверка:** Автоматически выполняется команда `docker logs`, и результат выводится в консоль Ansible.
*   *Примечание:* В файле `inventory.ini` указаны заглушки. Для запуска необходимо подставить реальный IP и путь к SSH-ключу.
